I.  Specific Process of Question Verification

SocialBench evaluates thread-scoped social-predicate aggregation. Each question requires you to (1) resolve a thread-defined scope, (2) apply a stated social predicate to every in-scope comment, and (3) aggregate the labels into a numeric answer. Verification at this portal is therefore three-axis joint agreement: chosen answer, enumerated in-scope node_ids, and per-comment include / exclude labels must all match the author.
Start Verification
  1. Click “Question Verification” in the sidebar to select an episode and question. The episode table at the top shows total_demand, #verified, and #pending_verification per episode. You can only pick tasks where #pending > 0; you cannot verify questions you authored yourself.
  2. Click Start Answering to download the episode JSON (thread DAG + per-node metadata: user · parent · content · domain-specific metadata) and load the question. The timer starts on click. Run through the seven question-quality checks below; if any fails, click Question does not meet requirements and write the rejection reason.
    1. Locator check. The stem points to a uniquely identifiable scope (largest sub-discussion, longest chain, etc.); no rank qualifiers (“2nd / 3rd”), no size hints (“the 27 replies”), no content glosses.
    2. Multi-comment aggregation. ≥ 15 comments in scope for count / ratio questions; 8–12 for ranking / odd-one-out. Single-comment retrieval is forbidden.
    3. Quantitative-relational answer. Options are quantities (e.g. {3, 5, 7, 9}) or relational tokens (rank, ratio, sequence, intersection). Descriptive option content is forbidden.
    4. Bare stem. No keyword cue for what to count, no instance lists. Concept-level disambiguation (“one-word reactions excluded”) is allowed; instance-level (“must name supply / demand / substitutes”) is not.
    5. Web-searchability. The stem is not solvable by a Google search using the stem’s own wording — no episode-title leak, no exact quotes > 5 contiguous words, no top-comment- content paraphrase.
    6. Predicate definition. The criterion_definition field gives a verifiable rule a third reader can apply to a comment without reading anything else.
    7. Distractor sanity. Each wrong option corresponds to a distinct error_type category (wrong_scope, inverted_stance, off_topic, etc.); none is fabricated content not present in the thread.
  3. After answering, you submit (a) your answer letter (A / B / C / D), (b) the in-scope node_ids you enumerated, and (c) your per-comment include / exclude annotation for every enumerated node. Joint agreement = (a) ∧ (b) ∧ (c) all match the author; an item with the right answer but a different scope or per-comment labelling does not count as agreeing.
  4. On the Reveal page you will see your submission alongside the author gold and the author’s criterion_definition. If they disagree, you classify the disagreement under the Phase-7 protocol: (a)  reviewer misread, (b)  library clause attaches, or (c)  question flaw → rewrite. Library clauses come from a pre-frozen criterion library; you select from a list, you do not write new clauses.
  5. The reward for a verified question is 25 CNY base + 50 CNY difficulty bonus when (i) the question is judged qualified and (ii) your verification time is between 5 and 15 minutes. If random sampling shows obvious cheating (instant answers, blindly choosing I don’t know this question, or matching gold without enumerating node IDs), all rewards for the account are revoked.

II.  Task Situation

After reading the requirements above, click Start Verification in the top-right corner. You will be assigned an episode + question and the timer will start.
